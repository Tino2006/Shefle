## 🚀 QUICK START - Secure Your Admin Panel in 3 Steps

### Step 1: Run the Database Migration (5 minutes)

1. Open your Supabase project at https://app.supabase.com
2. Navigate to SQL Editor
3. Create a new query
4. Copy and paste the entire contents of `supabase-rbac-migration.sql`
5. Click "Run" to execute

**What this does:**
- Adds `role` field to profiles table
- Creates admin helper function
- Updates all RLS policies
- Secures all tables and storage

### Step 2: Grant Admin Role to Yourself (1 minute)

In the same SQL Editor, run:

```sql
-- Replace with YOUR email address
UPDATE public.profiles 
SET role = 'admin' 
WHERE id = (SELECT id FROM auth.users WHERE email = 'YOUR-EMAIL@example.com');

-- Verify it worked
SELECT 
  u.email,
  p.first_name,
  p.last_name,
  p.role
FROM auth.users u
JOIN public.profiles p ON u.id = p.id
WHERE u.email = 'YOUR-EMAIL@example.com';
```

### Step 3: Test It! (5 minutes)

**Test 1 - Admin Access (Should Work)**
1. Login with your admin account
2. Navigate to `/admin` in your browser
3. ✅ You should see the admin dashboard
4. ✅ All admin features should work

**Test 2 - Regular User Blocked (Should Fail)**
1. Create a new test account OR use existing non-admin account
2. Login with that account
3. Try to navigate to `/admin`
4. ✅ You should be redirected to `/` with an error
5. ✅ Check URL shows `?error=unauthorized`

**Test 3 - No Auth Redirect (Should Redirect)**
1. Logout completely
2. Try to go directly to `/admin`
3. ✅ You should be redirected to `/login?redirect=/admin`
4. ✅ After login as regular user, still can't access admin

**Test 4 - API Protection (Should Block)**
Open browser console and try:
```javascript
// This should only work if you're an admin
fetch('/api/admin/stats')
  .then(r => r.json())
  .then(console.log);
```
- As admin: ✅ Should return stats
- As regular user: ❌ Should return 403 Forbidden

## 🆘 Troubleshooting

**Problem: Can't access admin even though I'm admin**
```sql
-- Check your role
SELECT id, first_name, last_name, role 
FROM public.profiles 
WHERE id = (SELECT id FROM auth.users WHERE email = 'your-email@example.com');

-- Fix: Set role to admin
UPDATE public.profiles 
SET role = 'admin' 
WHERE id = (SELECT id FROM auth.users WHERE email = 'your-email@example.com');
```

**Problem: Middleware not working**
- Restart your dev server: `npm run dev` or `yarn dev`
- Clear Next.js cache: `rm -rf .next` then restart

**Problem: RLS blocking admin queries**
- Verify `is_admin()` function exists:
  ```sql
  SELECT public.is_admin();
  ```
- Should return `true` when logged in as admin
