## 🔌 **2. API Routes Created**

### Authentication (`/api/auth/`)
- ✅ `POST /api/auth/signup` - Create new account
- ✅ `POST /api/auth/login` - Login with email/password  
- ✅ `POST /api/auth/logout` - Logout
- ✅ `POST /api/auth/forgot-password` - Request password reset
- ✅ `POST /api/auth/reset-password` - Update password
- ✅ `GET /api/auth/me` - Get current user

### Profile (`/api/profile/`)
- ✅ `GET /api/profile` - Get user profile
- ✅ `PUT /api/profile` - Update profile  
- ✅ `PUT /api/profile/password` - Change password
- ✅ `GET /api/profile/notifications` - Get notification preferences
- ✅ `PUT /api/profile/notifications` - Update preferences

### Brands (`/api/brands/`)
- ✅ `POST /api/brands` - Submit brand registration
- ✅ `GET /api/brands` - List user's brands
- ✅ `GET /api/brands/[id]` - Get specific brand

### File Upload (`/api/upload/`)
- ✅ `POST /api/upload` - Upload files to Supabase Storage

### Subscriptions (`/api/subscriptions/`)
- ✅ `GET /api/subscriptions/plans` - Get available plans
- ✅ `GET /api/subscriptions/current` - Get user's subscription

### Payments (`/api/payments/`)
- ✅ `POST /api/payments/create-checkout` - Create Stripe checkout session
- ✅ `GET /api/billing/history` - Get payment history

### Webhooks (`/api/webhooks/`)
- ✅ `POST /api/webhooks/stripe` - Handle Stripe events (subscriptions, payments)

### Contact (`/api/contact/`)
- ✅ `POST /api/contact` - Submit contact form
- ✅ `POST /api/contact/upload` - Upload contact attachments

## 🐛 **Troubleshooting**

### Supabase Connection Issues
- Make sure `.env.local` has correct Supabase credentials
- Check RLS policies in Supabase dashboard
- Verify storage buckets are created

### Stripe Issues
- Use Stripe test mode keys for development
- Set up webhook endpoint in Stripe dashboard: `http://localhost:3000/api/webhooks/stripe`
- Use Stripe CLI for local webhook testing:
  ```bash
  stripe listen --forward-to localhost:3000/api/webhooks/stripe
  ```

### File Upload Issues
- Check storage bucket permissions
- Verify file size limits (10MB max)
- Ensure correct file types (PDF for POA/license)

## 📚 **API Documentation**

All endpoints return JSON with the following structure:

**Success:**
```json
{
  "message": "Success message",
  "data": { ... }
}
```

**Error:**
```json
{
  "error": "Error message",
  "details": [ ... ] // Optional validation errors
}
```

**Authentication:**
- Most endpoints require authentication via Supabase session cookie
- Middleware automatically protects routes
- Use `auth.uid()` in RLS policies
